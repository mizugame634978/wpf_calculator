## 目的
`ICommand` と `RelayCommand` を理解する。ボタンのクリックをViewModelで処理する。

## 学習ポイント
- `ICommand` インターフェースの役割（`Execute`, `CanExecute`）
- `RelayCommand` の自作方法
- `CanExecute` でボタンを有効/無効にする方法
- CommandとBindingの組み合わせ

## 画面仕様

```
┌─────────────────────────┐
│  簡単な電卓             │
│                         │
│  ┌───────────────────┐  │
│  │              0    │  │  ← 結果表示（右寄せ）
│  └───────────────────┘  │
│                         │
│  [7] [8] [9] [÷]        │
│  [4] [5] [6] [×]        │
│  [1] [2] [3] [-]        │
│  [0] [C] [=] [+]        │
└─────────────────────────┘
```

## 機能要件
1. 0〜9 の数字ボタンで数値を入力できる
2. `+` `-` `×` `÷` で演算子を選択できる
3. `=` で計算結果を表示する
4. `C` で入力をリセットする
5. ゼロ除算（÷0）のときは `=` ボタンを無効にする（`CanExecute` を使う）

## 実装要件

### RelayCommand の自作（重要）
まず `RelayCommand` クラスを自分で作ること。これが `ICommand` の理解の核心。

```
ICommand インターフェースが持つメソッド:
- void Execute(object? parameter)   → ボタンが押されたときの処理
- bool CanExecute(object? parameter) → ボタンを押せるかどうか
- event CanExecuteChanged           → CanExecuteの結果が変わったことを通知
```

### ViewModel
- クラス名: `MainViewModel`
- `INotifyPropertyChanged` を実装する
- 以下のプロパティを持つ
  - `DisplayText` (string) — 画面表示用
- 以下のCommandを持つ
  - `NumberCommand` (ICommand) — 数字ボタン用（CommandParameterで数字を渡す）
  - `OperatorCommand` (ICommand) — 演算子ボタン用
  - `EqualsCommand` (ICommand) — = ボタン用
  - `ClearCommand` (ICommand) — C ボタン用

### View
- ボタンの `Command` プロパティにバインドする
- `CommandParameter` で数字や演算子の値をViewModelに渡す

### xaml.cs の制約（MVVM ルール）
`MainWindow.xaml.cs` に書いてよいのは `DataContext` のセットのみ。ロジックは書かない。

```csharp
public MainWindow()
{
    InitializeComponent();
    DataContext = new MainViewModel();
}
```

## ヒント
```csharp
// ICommand の最小実装（RelayCommand）
public class RelayCommand : ICommand
{
    private readonly Action<object?> _execute;
    private readonly Func<object?, bool>? _canExecute;

    public RelayCommand(Action<object?> execute, Func<object?, bool>? canExecute = null)
    {
        _execute = execute;
        _canExecute = canExecute;
    }

    public event EventHandler? CanExecuteChanged
        => CommandManager.RequerySuggested += value; // WPFが自動で再評価してくれる

    public bool CanExecute(object? parameter) => _canExecute?.Invoke(parameter) ?? true;
    public void Execute(object? parameter) => _execute(parameter);
}
```

<!-- CommandParameter の使い方 -->
<Button Content="7" Command="{Binding NumberCommand}" CommandParameter="7" />
